description: "All REST endpoints consumed by the AgroPulse frontend, grouped by resource, with request parameters, response shapes, and authentication header requirements."
---

The AgroPulse frontend communicates exclusively with the REST backend configured via `VITE_API_URL` (default `http://localhost:8080`). Every request is made through `BaseRepository` in `src/core/ApiService.ts`, which automatically injects authentication headers and handles JSON serialisation. This page lists every endpoint used by the frontend, grouped by resource.

## Authentication

All requests include the following headers when a user context is set via `setUserContext()`:

| Header | Type | Description |
|--------|------|-------------|
| `X-User-Id` | `string` | ID of the currently authenticated user. Present on all requests after login. |
| `X-Admin-Email` | `string` | Email of the user, included only when the user holds an admin role. Required by admin-only endpoints. |

<Note>Headers are injected automatically by `ApiService.ts`. You do not set them manually in application code unless you need a one-off override via `options.headers`.</Note>

## Base URL

```
http://localhost:8080
```

Override at build time or runtime by setting `VITE_API_URL` in your environment. See [Environment variables](/dev/environment).

  <Accordion title="Greenhouses">
    **`GET /api/greenhouses`**

    Returns all greenhouses the authenticated user has access to.

    <ResponseField name="greenhouses" type="GreenhouseDto[]" required>List of greenhouse objects.</ResponseField>

    ---

    **`GET /api/greenhouses/:id`**

    Returns a single greenhouse by ID.

    <ParamField path="path.id" type="number" required>Greenhouse ID.</ParamField>

    <ResponseField name="(root)" type="GreenhouseDto" required>The greenhouse object.</ResponseField>

    ---

    **`POST /api/greenhouses`**

    Creates a new greenhouse.

    <ParamField path="body.name" type="string" required>Display name.</ParamField>
    <ParamField path="body.location" type="string">Human-readable location.</ParamField>
    <ParamField path="body.description" type="string">Optional description.</ParamField>
    <ParamField path="body.deviceId" type="string">ESP32 device identifier to link.</ParamField>
    <ParamField path="body.latitude" type="number">GPS latitude.</ParamField>
    <ParamField path="body.longitude" type="number">GPS longitude.</ParamField>
    <ParamField path="body.photoUrl" type="string">Supabase photo URL.</ParamField>

    <ResponseField name="(root)" type="GreenhouseDto" required>The created greenhouse.</ResponseField>

    ---

    **`PUT /api/greenhouses/:id`**

    Updates an existing greenhouse (full or partial fields).

    <ParamField path="path.id" type="number" required>Greenhouse ID.</ParamField>

    Body accepts any subset of `GreenhouseDto` fields.

    <ResponseField name="(root)" type="GreenhouseDto" required>The updated greenhouse.</ResponseField>

    ---

    **`DELETE /api/greenhouses/:id`**

    Deletes a greenhouse.

    <ParamField path="path.id" type="number" required>Greenhouse ID.</ParamField>

    Returns `204 No Content`.
